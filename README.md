# UltimateJs

> A high-performance JavaScript framework with a Rust compiler core ‚Äî write once, render everywhere.

UltimateJs eliminates entire categories of boilerplate by making the infrastructure invisible. Three pillars drive the design:

| Pillar | Problem it solves |
|---|---|
| **Fluid Execution** | No more `"use client"` / `"use server"` annotations ‚Äî the compiler figures it out |
| **Semantic UI** | One component tree renders to Web, React Native, and Email simultaneously |
| **Zero-Fetch Sync** | No more `fetch()` ‚Äî state syncs over a CRDT-powered WebSocket delta stream *(Phase 4)* |

## Why UltimateJs?

Modern web development forces developers to make the same decisions repeatedly:

- "Is this component a Server Component or a Client Component?"
- "Should I `fetch()` this data or use a state manager?"
- "How do I reuse this UI in my mobile app? In my transactional email?"

UltimateJs answers all three questions at the framework level ‚Äî once ‚Äî so you never have to again.

## Pillar 1 ‚Äî Fluid Execution *(Phase 2, complete)*

The Rust compiler analyses every `.ultimate.tsx` file at build time using [SWC](https://swc.rs) and automatically produces two bundles ‚Äî no annotations required.

### How It Works

Given a single mixed file:

```tsx
// components/UserDashboard.ultimate.tsx

import { db } from './db';                    // ‚Üê server trigger (DB import)

export async function getUser(id: string) {   // classified: ServerOnly
  return db.user.findUnique({ where: { id } });
}

export function UserCard({ userId }: { userId: string }) {
  window.analytics.track('view');             // ‚Üê client trigger (browser global)
  return <button onClick={() => getUser(userId)}>Load User</button>;
}
```

The compiler produces two outputs automatically:

**`.ultimate/UserDashboard.server.js`** ‚Äî contains `getUser`, strips `UserCard`

**`.ultimate/UserDashboard.client.js`** ‚Äî contains `UserCard`, replaces `getUser` with a type-safe RPC bridge:

```ts
// Auto-generated by the UltimateJs compiler ‚Äî never written by hand
export async function getUser(id: string) {
  return __ultimate_rpc('/api/__ultimate/getUser', { id });
}
```

### Classification System

The Rust `Classifier` runs two passes over every declaration in the file:

| DeclKind | Meaning | Example trigger |
|---|---|---|
| `ServerOnly` | Uses server resources | `process.env`, DB imports, `node:*` modules |
| `ClientOnly` | Uses browser APIs | `window`, `document`, `localStorage` |
| `Shared` | Pure logic, safe anywhere | Math, string utilities |
| `BoundaryCrossing` | Server fn called from client | Generates RPC stub automatically |
| `Mixed` | Both triggers in same fn | Flagged as a compile error |

### Server Triggers (detected by `SecretScanner`)

- `process.env.*` ‚Äî environment variable access
- Database imports: `prisma`, `drizzle`, `mongoose`, `pg`, `mysql2`, `better-sqlite3`, and more
- Node.js protocol imports: `node:fs`, `node:crypto`, etc.

### Client Triggers (detected by `CapabilityScanner`)

- Browser globals: `window`, `document`, `localStorage`, `sessionStorage`, `navigator`, `location`
- DOM event APIs, `fetch` in browser context

### Vite Plugin

`@ultimatejs/vite-plugin` intercepts every `.ultimate.tsx` / `.ultimate.tsx` file and routes it through the Rust binary via a `stdin ‚Üí stdout` JSON bridge. Results are cached per file ID and cleared on HMR update.

```ts
// vite.config.ts
import { ultimatePlugin } from '@ultimatejs/vite-plugin';

export default {
  plugins: [ultimatePlugin()()],
};
```

## Pillar 2 ‚Äî Semantic UI *(Phase 3, complete)*

Four semantic primitives ‚Äî `Stack`, `Text`, `Action`, `Input` ‚Äî describe *what* a UI element is, not *how* to render it. Each render target (`@ultimatejs/web`, `@ultimatejs/native`, `@ultimatejs/email`) maps them to its own native elements.

### Design Token System

All spacing and color values use a design-token-shaped type system ‚Äî no magic strings.

**`SpaceValue`** ‚Äî maps to Tailwind's 4px-unit convention:

| Token | px equivalent |
|---|---|
| `0` | 0px |
| `1` | 4px |
| `2` | 8px |
| `4` | 16px |
| `6` | 24px |
| `8` | 32px |
| `"16px"` | 16px (passthrough) |
| `"50%"` | 50% (passthrough) |

**`ColorToken`** ‚Äî semantic color names resolved by the active theme:

`background`, `surface`, `border`, `primary`, `primary-fg`, `secondary`, `secondary-fg`, `success`, `warning`, `danger`, `muted`, `muted-fg`

On **web** these resolve to CSS custom properties (`var(--ultimate-primary)`), letting you theme the entire UI by setting variables on `:root`. On **native** and **email** they resolve against a built-in `DEFAULT_THEME` palette (hex strings).

### The Three Render Targets

The `UltimateRenderer<TNode>` contract enforces that every target implements all four primitives:

```ts
interface UltimateRenderer<TNode> {
  Stack:  (props: StackProps)  => TNode;
  Text:   (props: TextProps)   => TNode;
  Action: (props: ActionProps) => TNode;
  Input:  (props: InputProps)  => TNode;
}
```

#### `@ultimatejs/web` ‚Äî `TNode = ReactElement`

React components with inline CSS styles. No Tailwind dependency ‚Äî works in any React app out of the box. Colors use CSS custom properties so the theme is overridable at the `:root` level.

```tsx
import { Stack, Text, Action, Input } from '@ultimatejs/web';

function LoginForm() {
  return (
    <Stack direction="column" gap={4} padding={6} background="surface" radius="md">
      <Text variant="title">Sign in</Text>
      <Input type="email" fieldLabel="Email" placeholder="you@example.com" />
      <Input type="password" fieldLabel="Password" placeholder="‚Ä¢‚Ä¢‚Ä¢‚Ä¢‚Ä¢‚Ä¢‚Ä¢‚Ä¢" />
      <Action variant="primary" fullWidth>Continue</Action>
    </Stack>
  );
}
```

#### `@ultimatejs/native` ‚Äî `TNode = ReactElement`

React Native components. `SpaceValue` resolves to dp numbers, `ColorToken` resolves against `DEFAULT_THEME`. The `as` prop (HTML element override) is silently ignored. `href` opens via `Linking.openURL`. Loading states use `ActivityIndicator`.

```tsx
import { Stack, Text, Action, Input } from '@ultimatejs/native';

// Same primitives ‚Äî identical props ‚Äî renders natively on iOS/Android
function LoginForm() {
  return (
    <Stack direction="column" gap={4} padding={6} background="surface" radius="md">
      <Text variant="title">Sign in</Text>
      <Input type="email" fieldLabel="Email" placeholder="you@example.com" />
      <Action variant="primary" fullWidth>Continue</Action>
    </Stack>
  );
}
```

#### `@ultimatejs/email` ‚Äî `TNode = string`

Pure HTML string output ‚Äî no React. Layout uses `<table>` elements (Outlook-safe). All styles are inline. Interactive props (`onPress`, `onChange`) are silently ignored. Includes a `wrapDocument()` utility for full MSO-safe email documents.

```ts
import { Stack, Text, Action, wrapDocument } from '@ultimatejs/email';

const body = Stack({
  padding: 6,
  background: 'surface',
  children: [
    Text({ variant: 'heading', children: 'Reset your password' }),
    Text({ variant: 'body', children: 'Click the button below to reset your password.' }),
    Action({ variant: 'primary', href: 'https://example.com/reset', children: 'Reset Password' }),
  ],
});

const html = wrapDocument(body, {
  title: 'Password Reset',
  previewText: 'Reset your UltimateJs account password',
});
// ‚Üí full MSO-safe HTML document, ready to send
```

## Pillar 3 ‚Äî Zero-Fetch Sync *(Phase 4, complete)*

Replace `fetch()` calls with a CRDT-powered WebSocket delta stream compiled to WebAssembly. No manual data fetching ‚Äî state syncs automatically across all connected clients.

```tsx
// What you'll write
const [user, update] = useSync('users', userId);
update('name', 'Alice');   // optimistic + synced to all peers instantly

// What UltimateJs does invisibly
// ‚Üí opens a WebSocket to ws://<host>/sync/users/<id>
// ‚Üí receives binary "delta" patches (Automerge CRDT format)
// ‚Üí applies them locally via WASM ‚Äî deterministic conflict resolution
// ‚Üí rolls back optimistically if the server sends a rejection frame (0xFF)
```

**Task 4.1** ‚Äî `@ultimatejs/crdt`: Automerge (Rust) ‚Üí WASM via `wasm-pack`; `CrdtDoc` API via `wasm-bindgen`
**Task 4.2** ‚Äî `useSync` hook in `@ultimatejs/core`: connects React to WASM CRDT state over WebSocket
**Task 4.3** ‚Äî `@ultimatejs/sync-server`: binary WebSocket server, per-document rooms, GC on disconnect
**Task 4.4** ‚Äî Optimistic rollback: `confirmedBytesRef` snapshots + 0xFF rejection frame protocol

## Pillar 4 ‚Äî Sidecar Worker *(Phase 5.1, complete)*

Third-party tracking scripts (analytics, tag managers, ad pixels) block the main thread and degrade Core Web Vitals. `@ultimatejs/sidecar` offloads them to a Web Worker ‚Äî they run off the main thread and access the DOM via an async proxy.

```html
<!-- Before: blocks your main thread -->
<script src="https://www.googletagmanager.com/gtag.js"></script>

<!-- After: runs in a Web Worker -->
<script type="text/ultimatejs" src="https://www.googletagmanager.com/gtag.js"></script>
```

```ts
import { initSidecar } from '@ultimatejs/sidecar';

// Call once at app startup ‚Äî picks up all <script type="text/ultimatejs"> tags
// and watches for any added dynamically (e.g. by a tag manager)
const sidecar = initSidecar({ workerUrl: '/sidecar.worker.js' });

// Clean up when done
sidecar.destroy();
```

### How It Works

```
Main thread                          Web Worker
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ    ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
<script type="text/ultimatejs">  ‚îÄ‚îÄ‚ñ∫ fetch(url) ‚Üí new Function(source)
                                     runs script with proxy window/document
                                     ‚Üì
window.dataLayer.push(‚Ä¶)  ‚óÑ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ { type:'call', path:['dataLayer','push'], args }
return result  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∫  Promise resolves ‚Üí script continues
```

- The Worker fetches each script's source and runs it via `new Function('window', 'document', ‚Ä¶, source)` ‚Äî our proxy objects are injected as `window` and `document`
- DOM property reads return sub-proxies lazily; function calls send a `call` message and return a `Promise`
- A `MutationObserver` catches scripts injected after page load (e.g. GTM auto-injecting pixels)
- Native Worker globals (`fetch`, `setTimeout`, `Math`, etc.) bypass the proxy ‚Äî no overhead

## Pillar 5 ‚Äî Nexus Inspector *(Phase 5.2, complete)*

`@ultimatejs/inspector` is a zero-dependency browser overlay for development that shows exactly which components are running on the server, which run on the client, and which cross the boundary ‚Äî directly on your rendered page.

```ts
import { initInspector } from '@ultimatejs/inspector';

// Drop this in your dev entry point
const inspector = initInspector();

// Toggle with a keyboard shortcut
document.addEventListener('keydown', (e) => {
  if (e.altKey && e.key === 'i') inspector.toggle();
});

// Clean up on HMR dispose
if (import.meta.hot) {
  import.meta.hot.dispose(() => inspector.destroy());
}
```

Mark components in the DOM using the `data-ultimate-kind` attribute (set automatically by `@ultimatejs/web` or the compiler):

```tsx
// @ultimatejs/web adds this automatically based on compiler output
<div data-ultimate-kind="server" data-ultimate-name="UserCard">‚Ä¶</div>
<div data-ultimate-kind="client" data-ultimate-name="Button">‚Ä¶</div>
<div data-ultimate-kind="boundary" data-ultimate-name="useUserData">‚Ä¶</div>
```

### What you see

| Kind | Colour | Meaning |
|---|---|---|
| `server` | üîµ Blue | Runs only on the server ‚Äî no browser APIs |
| `client` | üüÝ Orange | Runs only in the browser ‚Äî uses DOM/window |
| `shared` | üü¢ Green | Safe on both sides ‚Äî pure logic |
| `boundary` | üü£ Purple | Server fn called from client ‚Äî RPC stub generated |
| `mixed` | üî¥ Red | Error ‚Äî uses both server and client triggers |

A floating panel in the bottom-right corner shows a live count of each kind. It auto-updates via `MutationObserver` as components mount and unmount.

## Prerequisites

- [Node.js](https://nodejs.org) 20+

## Getting Started

```bash
# 1. Clone and install JS dependencies
git clone https://github.com/ChandanBose666/UltimateJs.git
cd UltimateJs
pnpm install

# 2. Build the Rust compiler (required before first use)
cd packages/compiler
cargo build --release
cd ../..

# 3. Run tests
cd packages/compiler && cargo test    # Rust unit tests
cd packages/email   && npx jest       # Email renderer tests (40 tests)

# 4. Start the dev server
pnpm dev
```

> **Windows note:** `cargo` is not on Git Bash's PATH by default. Either run Rust commands in Windows CMD, or add `export PATH="/c/Users/$USER/.cargo/bin:$PATH"` to your `~/.bashrc`.
